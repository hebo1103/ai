# 测试技能

为代码的函数、类和模块生成高质量的单元测试。

## 指令

1. **分析目标代码**：阅读并理解要测试的函数/类，包括输入、输出、边界情况和依赖关系。

2. **识别测试用例**：
   - 正常/成功路径情况
   - 边界条件（空输入、最大值、零、空值）
   - 错误/异常情况
   - 特定逻辑的边界情况

3. **遵循以下规则生成测试**：
   - 使用项目现有的测试框架（从依赖或现有测试文件中检测）
   - 遵循现有的测试文件命名约定
   - 每个测试尽可能只包含一个逻辑断言
   - 使用描述性的测试名称：`test_<函数>_<场景>_<预期结果>`
   - 模拟外部依赖，不要模拟被测单元

4. **测试结构模板**：

```
Arrange（准备） - 设置测试数据和模拟
Act（执行） - 调用被测函数
Assert（断言） - 验证预期结果
```

## 框架检测顺序

1. 检查现有测试文件的导入和模式
2. 检查 `package.json` / `requirements.txt` / `pom.xml` / `go.mod` 中的测试依赖
3. 按语言默认选择：
   - Python: pytest
   - JavaScript/TypeScript: Jest
   - Java: JUnit 5
   - Go: testing (标准库)

## 检查清单

- [ ] 覆盖正常路径
- [ ] 覆盖边界条件
- [ ] 覆盖错误/异常情况
- [ ] 测试独立且可重复
- [ ] 测试不依赖执行顺序
- [ ] 外部依赖已模拟

## 输出格式

提供可直接运行的测试文件，包括：
- 所有必要的导入
- 测试类/describe 块
- 独立的测试方法
- 简要注释说明每个测试的目的
